# Localnet example

This example lets you test `mppx-solana` against a Solforge or Solana localnet.

## What it does

- starts a Bun server with a paid `GET /paid` route
- challenges the client with the custom `solana` MPP payment method
- sends a native SOL localnet payment from the client
- verifies the transaction and returns a receipt

## Prerequisites

- a localnet recipient wallet address
- a localnet payer wallet with enough SOL for the payment and fees
- Bun installed

Solforge defaults in `sf.config.json` expose local RPC on port `8899`, so the examples now default to:

```bash
http://127.0.0.1:8899
```

You can fund a wallet on localnet with the Solana CLI:

```bash
solana airdrop 2 <wallet-address> --url http://127.0.0.1:8899
```

## Start the server

```bash
MPP_SECRET_KEY=dev-secret \
EXAMPLE_RECIPIENT=<recipient-wallet-address> \
EXAMPLE_CLUSTER=localnet \
EXAMPLE_RPC_URL=http://127.0.0.1:8899 \
EXAMPLE_AMOUNT=1000000 \
bun run example:server
```

Optional environment variables:

- `PORT` default `3000`
- `EXAMPLE_CLUSTER` default `localnet`
- `EXAMPLE_COMMITMENT` default `confirmed`
- `EXAMPLE_CURRENCY` default `solana:native`
- `EXAMPLE_DECIMALS` default `9` for native SOL and `6` otherwise
- `EXAMPLE_MEMO` default autogenerated

## Run the client

You can provide the payer secret key either as:

- a JSON array: `[12,34,...]`
- or a base58-encoded secret key

```bash
EXAMPLE_PAYER_SECRET_KEY='[12,34,...]' \
EXAMPLE_CLUSTER=localnet \
EXAMPLE_SERVER_URL=http://localhost:3000/paid \
bun run example:client
```

Optional environment variables:

- `EXAMPLE_CLUSTER` default `localnet`
- `EXAMPLE_RPC_URL` overrides the default cluster RPC URL

### Using cloned USDC or another Solforge token

If Solforge cloned a token mint into localnet, you can point the server at it:

```bash
EXAMPLE_CURRENCY=EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v \
EXAMPLE_DECIMALS=6 \
bun run example:server
```

The library supports that mint address, but your payer wallet still needs balance in that token.
For fully automatic tests, the included `example:e2e` script still uses native SOL by default.

## Run the end-to-end script

This script generates fresh payer and recipient keypairs, airdrops localnet SOL to the payer,
starts the example server, performs the payment, verifies the receipt, and shuts the server down.

```bash
EXAMPLE_CLUSTER=localnet \
bun run example:e2e
```

Optional environment variables:

- `PORT` default `3020`
- `EXAMPLE_CLUSTER` default `localnet`
- `EXAMPLE_RPC_URL` overrides the default cluster RPC URL
- `EXAMPLE_SOL_AMOUNT_LAMPORTS` default `1000000`
- `EXAMPLE_MEMO` overrides the autogenerated memo
- `MPP_SECRET_KEY` default `dev-secret`

## Expected result

The client should print:

- the paid route response JSON
- the MPP payment receipt
- the recipient balance change after payment

The server should log the paid route configuration including the memo and recipient.
